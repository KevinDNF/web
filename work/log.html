HTML Structure

Head

Every html document has a head section, in which important information is defined like title, meta tags and links to stylesheets/scripts.

This informatin is not showned in the page but it is used for other necessary actions the webpage will perform. 


			
<meta charset="utf-8">
<meta name="author"content="UP849868">
<meta name="viewport"content="width=device-width,initial-scale=1.0">
<link rel="shortcut icon" type="image/png"href="../images/logoInverted.png">
<link rel="stylesheet"href="../style.css">

The meta tag are used for metadata.

The link tag are used to link other files like stylesheets or javascript.

The charset is the encoding used on the page. In this case I'm using standard utf-8

Viewport is used for compatibility with mobile devices or small screens. The viewport meta tag has tons of functionality and will require further research to explain it fully.

the rel="shortcut icon" is used to display a favicon image in the tab's browsers.

Body - Semantic Markup

Its important that the body of the html document follows good semantic markup to ensure that the webpage can be read by screenreaders. 


<nav>
  <a>
</nav>
<article>
  <header>
  </header>
  <section>
  </section>
  <footer>
  </footer>
</article>

A well defined website will have a good semantic structure meaning each part of the website has its corresponding tag. So the header elements will be enclosed in a header tag, the main will be enclosed in the main.

Articles are used for blog post like documents and other form of repetitive content.

Sections are sections inside articles or just the main.

Main is where the main content of the webpage goes.

Footer and Header is as standard documents have header and footer. The important part of these elements is that the html document will have a header and footer and articles will have a header and footer meaning we can utilized them in many different other semantic tags.

Nav is used for navigation of the website. This is not only the top navigation bar, but can be used for many forms of navigations throught out the webpage. In this website I have 2 sets of navigation; the top sticky bar and the sliding one.

<b> vs <strong>

Due to the introduction of semantic markup we now have two ways of adding boldness to text but they are completly different in how they are meant to be implemented. The b tag is used as usual, for visual boldness but Strong is used to mark importance to text in a webpage level so that web crawlers or bots can use this tags to find what is important to the webpage.

Media Elements

Pictures and images

On HTML5 the introduction of the picture tag was quite a big change to the image world, since before using the img tag was the only way of implementing images.

The img tag is used to add an image to a webpage. This can be of any image format and is essential that it contains a good descriptive alternative tag. This alternative text will be displayed if the image fails to load and has been a html requirement to avoid having unwelcoming sites to screenreaders.

Being able to add images is great but with the introduction of the Picture tag we can do a lot more. With the picture tag we can add many sources to a single image meaning that if one of them fails to load because it's not supported or other reason then a second source will load. This can be achieved using the type tag inside the souce tag.

Instead of just defining different sources in case it fails there is also the feature of using media queries within the picture tag so that a different image loads at a specific screen size or other queries. 



<picture>
  <source srcset="images/icons/github.svg"type="image/svg+xml" >
  <img src="images/icons/github.png"alt="github">
<picture>

The crucial part of the picture tag is that it must contain either a media query or a type tag.

CSS 3

CSS is the styling of the webpage, how it looks and how it reflows. There are two main ways of implementing css.

Inline styling

Using the style attribute inside a tag can add style to a specific element. But by doing this we are styling each element specifically, so really big documents will require a stupid amount of time to correctly style. 



<p style:background-color="red";>Some text

Which is why it is not recommended to use.

External Style Sheet

We can style every element in a page by using an external style sheet, which is usually a something.css containing css. 



<p>Some text
		


p{
  background-color:red;
}

This is really usefull as we can have 1 css file linked to many html pages and have a consistent web design. So if we want to change one thing on every page we only have to edit 1 file.

To link an style sheet file to an html document we have to use the following line in our head section. 



<link rel="stylesheet"href="../style.css">

Where "../style.css" is the path to the file.

CSS Selectors

CSS works by selecting different tags and styling them as we wish. 



<p> 
  Text 1
</p> 

<main>
  <p> 
   Text 2 
  </p> 
</main>




p{
  color:red;
}

main p{
  color:blue;
}

In this example we have a p tag inside main and another one outside of main. If we want to access all of the p tags we can just use a simple p{ } which will apply the styling to all the p tags in the page.

But we can be more specific with the way to select what we want to style. In this case if we only want to style Text 2 which is inside the main tags we can use main p { } to select all the p tags inside the main tag.

Media Queries

When smartphones became and standard item for evey user to carry around websites struggled to resize correctly to every smartphone and opted to have a seperate .mobile version just to adapt to smartphones. This was later improved with the addition of media queries.

We can use media queries in css to change the style when a certain condition occurs. The most common one and the one I will be using through out this webpage is the screen attribute, which checks the size of the screen. 



@media screen and (max-width:520px){
  nav{
    padding: 9px;
  }
}

In this example when the screen is less than 520px will change the navigation padding to 9px.

The good thing about media queries is that you can have lots of different css selectors inside of the media query so technically you can have a completly new webpage for mobile users.

Flex-Box

Flex-box is a super usefull way of formatting a webpage and ensure that is reactive to screen size. The idea behind flex box is that you can apply it to a set of elements in a container so that they fill the container to a set of rules.

The first step to use flexbox is have the elements you want to fill the screen inside of a container. This will make every element fill the space diagonally.

By changing the flex-flow to column the space will be filled horizontally. 



body{
  display:flex;
  flex-flow:column;
}

body > *{
  flex:1 100%;
}

In this example I'm assigning flex to the body as my container and then giving a flex value of 1 and flex basis of 100% to every element inside of body so that it fill the whole area.

I have utilize media queries in combination with flexbox to ensure that this website is reactive to all screen sizes. Try resizing this page and see what happends to the navigation.
